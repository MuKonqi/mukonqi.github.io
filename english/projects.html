BetterXP (discontinued)

I started to build BetterXP based on Terminalden kurtulun. This was both out of curiosity and to remove the logo from the home page of the Terminalden kurtulun project. The first BetterXP releases contained my own improvements. But in the later versions under 1.x I added new support to those improvements, namely Fedora Linux and Solus support. One of the reasons for adding support was that I was a distrohopper at the time.
After that, 2.0 came out and that was short-lived. I changed the colors there and I made some improvements for GUI. And finally it died because the code was very, very complicated and I wanted a better name. BetterXP was for better, MetterXP is for maximum.

MetterXP


My best project thanks v3.0. To talk about advenced tools:

  • Application store
  • Desktop Environment and Windows Manager store
  • Package Manager store
  • Configuring softwares
  • Configuring .bashrc file
  • Opening file managers with root rights
  • Inform about some GNU/Linux distributions
  • Theme system
    • Seventeen theme
      • From developer (me)
      • Basic color themes
      • People's themes
    • Customizable theme
  • And more!

The most important innovation of v1.0 was actually for me: The module system. In this way, each module had its own module and this made it very easy for me to make new updates.
The biggest advantages of version v2.0 were the theme system and the addition of English language. But the biggest disadvantage of this theme system was that the settings were stored in the /usr/local/bin/metterxp/settings folder. So only 1 setting on a computer, if there were 2 users they had to use it together.
With version 3.0, the information is now stored in the /home/username/.by-mukonqi/metterxp/ folder. This means happiness and now everyone can enjoy their own taste and their own language! Also for all the innovations: Click here.

Both BetterXP and MetterXP had their own installers (not wizard), but I wanted to make a global installer wizard, so I released the Apiutaller wizard. Apiutaller also made my own work easier in many of my subsequent projects.
With v2.0, Apiutaller got rid of childish and unnecessary sympathetic expressions. Also reinstallation feature added. For all the changes: Click here.

Mukander, Mukotes, Muktart (Mukotes is my favorite)

The purpose of the Mukander project is to launch commands. It can start with sudo if the user wants. Obviously it can never be a terminal as it is.

Of these three, this is the one that really works for me. It can create, delete and edit notes. That's enough for me.

What Muktart did: Run ls command for looking /usr/share/applications directory's items later lists the applications and launches them on user's demand (normally or with sudo).
By the way, people start applications like this before start menu was invented (!).
